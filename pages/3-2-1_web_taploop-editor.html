Taploop Editor

This is a sub-project of Taploop, a platform sharing the experience of games, and this is its editor, based on markdown syntax with live-preview.

What is Taploop Editor designed for?

Taploop is something like "Let's Play", but, in the form of writing, just like Medium. The goal we want to do this, is to find some games really good on its art, script, or music, etc.. This kind of games, take Pillars of the Earth for example, does not seems so fascinating on "Let's Play" living show, but they do deserve literary reviews written by players. And I believe, game is already another form of stage art, just like opera, drama, and movie.


Interface

The main page of Taploop Editor is consisted of 3 parts: nav-list, side-nav, and edit space, in which side-nav is divided into 2 parts: default folders list, and file list.

The two file management panels will be folded if the device-width is down to 1280px. In this case it can be open by click/tap the "ï¹¤" icon.


Basic style

The style of this little component actually dominates the whole project. from buttons to dropdowns.

By the way, I like to outter borders, because it cause no illusions when the box is floating upon other elements, or upon its own shadow.

Hover style

Because some hovable components has their own big icon, the z-index of "hover-grey" is designed between the icon and the text. Notice that the "hover grey" here is not background-color, because I want the color of the text changes a little bit while user hover the components.

Here I made a demo by very basic html and CSS. It is quite similar to the hover style I wanted, but still there is a little difference: In the demo, when you hover the column, the "hover grey" dose not actually "cover" the text, so the blue "#" here dose not change at all. But in my Taploop Editor.sketch file, you will see the text is covered by a grey div. Well, it dosen't matter to most of people I guess...

Writing mode

Basicly, Taploop Editor is a live-preview markdown editor. Compared to tranditional markdown editor, as I have been told by our engineers, it is much harder to develop. And personally, I prefer to use some traditional markdown editor like MWeb or Atom. But, well, I have no choices, because I wanted people to write something first, but not be scared by syntax -- although it's easy...

Yeah, you might be familiar with this dropdown already. It is acctually followed the dropdown in Notion. To be honest, I tried several other plans for designing for this funtion, but, yet, I have no better idea but this. So, as an UI designer, I just changed some details. One is, I put the syntax hint right below the name of the elememts. I hope people can get familiar with markdown while using this editor, since the syntax is extremely easy to learn.


Source code mode

In sourse code mode, the markdown tag will be highlighted in the brand-color. Formats in source code mode won't be full-displayed, which means, the size will be the same. I designed as this because I want people know clearly about which mode he/she is using right now, and still, using minimal style change, to distinguish the different formats.

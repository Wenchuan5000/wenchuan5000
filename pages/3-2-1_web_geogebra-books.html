GeoGebra Books

In this file, you will see an imaginary project designed based on GeoGebra, an online math tool. The introduction of the file might a little bit long, because it is an UI/UX design file and also a new function sheet. Hope enjoy!

Why did this?

This project is imaginary, totally, yet. It is a rebound design for GeoGebra.com, whose one of the main tool GeoGebra Classic 6 is the most important tool during my last year of math learning. I like it so much, because it allows me to visualize the functions and definitions in coordinates so quickly, which means it allows you to feel mathematics intuitively, which is an extremely crucial step to understand mathematics.

By any means, GeoGebra Classic 6 and some other GeoGebra tools is much more powerful than most of online tools for hobbies, and it can deal with much more complicated problems than Desmos can. But, maybe because of the complexity, GeoGebra also has a lot of problems I've found since the first time I use it. Firstly, it is unstable on outputs. Secondly, its UX is a nightmare, compared to some other tools like Sketch whom I am using every. Thirdly, the UX of math books and activities is also not as good as it should be, on both ends of reading and editing.

Well, to deal with the first problem, I need many new knowledges far beyond what I've known yet. The problem No.2 is a problem of UX, but to solve it, rebuilding might be necessary -- I am still expecting to send my suggestion to them, but maybe it's not time yet.

I think, what I can do for GeoGebra yet is the problem No.3, the UI/UX on GeoGebra Books. Because as an UI/UX designer, I believe I am familiar with different types of editors, including rich text editors like Pages, markdown editors like MWeb, fake-markdown editors like Notion, and coding tools like Atom, etc., and am much more familiar with these, and with how a math-writer needs compared to most of designers. By the way, I am thinking about a better way to write/read math books every day, because I love math. That is why I think I should try under a math tool I am already somehow familiar with.

In my GeoGebra Books, the first thing I want to do, is to change the rich text editing mode into a simpler mode. I decided to design based on Markdown + LaTeX syntax, mainly because it is the "language" I most familiar with, and is light and easy enough to handle with for math-writes.

How to create a new book

Markdown + LaTeX

Now, let's forget the design, and talk about how my GeoGebra Book works. Basically, I used a very popular editing style now days, which is based on Markdown + LaTeX, and can be live-previewed.

This is designed for people like me hate to use mouse click too much for just one simple equation. If the real GeoGebra is likely to use this style of editing, I believe I can write a whole math book in GeoGebra -- I am serious!

But GeoGebra Books is not just another markdown editor, it is for science! So, here we go.

Create a new book

By clicking the big button "+ New Activity" on activity table on chapters' page, an untitled empty page (activity) is opened. Now you can write what you want to write.

As you can see, the elements which are allowed to use in introduction includes basic markdown elements and LaTeX formula.

You can upload your book cover in three ways: upload an image, embed an URL, or you can just pick from the gallery, which contains some inaccurate but universally usable covers.

For example, let's link a image URL.

By clicking the "setting" column on side-nav, you can edit further details of you book.

Chapter editor

Chapter editor is also an essential function in GeoGebra Books and my GeoGebra Books. I made some change here. In my "version", chapter editor looks very familiar with page editor (activity editor). Such that the introduction of the chapter can be easily edited on the current page.

By clicking the button "+ New Chapter" on side-nav, a default untitled chapter is opened.

On the editing area, there are three parts: chapter title (default string: Untitled Chapter), introduction (default: empty), activities table (default: empty).

Page editor (activity editor)

By clicking the big button "+ New Activity" on activity table on chapters' page, an untitled empty page (activity) is opened. Now you can write what you want to write.

By using Markdown + LaTeX, you can free from the mouse-clicks, and easy write formula within a pair of $ for inline formula and within a pair of $$ for math block.

And just like what you can do in Notion, by clicking the "+" icon, or press the hot key Command + / to activate a dropdown.

In this dropdown, you can select the traditional elements in GeoGebra. And first thing first, the GeoGebra Applet.

For example, if you choose to create a new applet, the browser will automatically send you to a new tab of GeoGebra Classic 6, and there leaves a blank embed applet in the previous page, to ready to receive the data from GeoGebra Classic 6.

Once you saved the applet on the page of GeoGebra Classic 6, the blank embed applet will receive the data, and show the figure you have edited.

Then, you can resize the applet, and add caption to the applet.

Book reader

When the book is published, or shared with link, now people can read it, as well as you can read a private book by yourself.

As you can see, the side-nav has changed, and became a contents tree of the book, in which the activities are displayed as unordered lists.

Just below introduction of the book, a table of contents will be generated. They are underlined because they are actually links.

The table of activities will be no longer changeable on reading mode.

This is the page of an activity. The figure below "Definition 1" is an image element. Notice, on the nav-bar, the path of the content is divided by "/"

The embed applet is no longer changeable neither, despite of the interactive things the author allows you to change, like the scale of the view, or some changeable variables.
